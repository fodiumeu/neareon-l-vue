# V1.0.0 RC1 Checkliste

## 1. Zweck

Diese Checkliste bereitet den aktuellen NEAREON Laravel/Vue-Stand als V1.0.0 RC1 vor. Ziel ist eine belastbare MVP-Demo, interne Prüfung und anschließende Testphase mit ausgewählten Personen. `v1.0.0-rc.1` ist noch keine Produktivfreigabe, sondern ein Release Candidate für MVP-Demo, interne Prüfung und Testphase.

## 2. Aktueller Referenzstand

- Referenz-Tag vor RC: `v0.10.66-candidate-smoke-fixes`
- Geplanter Release-Candidate-Tag: `v1.0.0-rc.1`
- Späterer finaler MVP-Tag nach bestandener Testphase: `v1.0.0`
- Vor dem Release-Candidate-Tag erneut `git status`, Tests und Build prüfen.
- Der `DemoSeeder` ist vorhanden, optional und wird manuell ausgeführt.
- Demo-Login für Präsentationen: `demo.fodi@neareon.test` mit Passwort `neareon-demo`.

## 3. DemoSeeder Vorbereiten

Standard-Demo-Daten einspielen:

```bash
php artisan db:seed --class=DemoSeeder
```

Optionaler frischer lokaler Start, nur wenn lokale Daten gelöscht werden dürfen:

```bash
php artisan migrate:fresh --seed
php artisan db:seed --class=DemoSeeder
```

- Demo-Account: `demo.fodi@neareon.test`
- Passwort: `neareon-demo`
- Die Demo-Daten sind synthetisch.
- Der DemoSeeder ist nicht für Produktion gedacht und bricht in `production` ab.

## 7. Kernflows Für MVP-Test

- [ ] Mitglieder entdecken → Profil ansehen → Folgen oder Kontaktanfrage senden.
- [ ] Kontaktanfrage ansehen → annehmen oder Status prüfen.
- [ ] Nachrichten öffnen → Unterhaltung ansehen → Nachricht senden.
- [ ] Gruppen entdecken → Gruppe ansehen → beitreten oder Anfrage senden.
- [ ] Meine Gruppen → Gruppe ansehen → Mitglieder/Management prüfen.
- [ ] Events entdecken → Event ansehen → teilnehmen oder Teilnahme anfragen.
- [ ] Meine Events → eigenes Event ansehen → bearbeiten → absagen → wieder aktivieren.
- [ ] Benachrichtigung öffnen → Zielseite prüfen.
- [ ] Profil bearbeiten → Sprachen und Interessen prüfen.

## 8. Erwartete Demo-Daten

Nach dem DemoSeeder sollten ungefähr vorhanden sein:

- mehrere Demo-User mit `@neareon.test`
- Profile mit Regionen, Sprachen und Interessen
- Kontakte, Follows und Kontaktanfragen
- Gruppen mit `public`, `request` und `private`
- Events mit `public`, `request`, eigenen und abgesagten Zuständen
- eine Demo-Konversation
- interne Benachrichtigungen
- keine vergangenen Events in Events entdecken

## 9. Bekannte MVP-Grenzen

- Business- und Unternehmensprofile
- Monetarisierung
- Payment
- bezahlte Gruppen oder Events
- Organisationsrollen und Mitarbeiterverwaltung
- echte Push-Notifications
- finaler Produktiv-Mailversand und vollständige E-Mail-Flows
- App-Store-Veröffentlichung
- Produktivbetrieb
- rechtliche Finalisierung von AGB, Datenschutz und Impressum
- Skalierungs- und Performance-Finetuning
- finale Moderations- und Admin-Prozesse

## 10. V1.0.0-RC1-Entscheidung

V1.0.0 RC1 ist bereit, wenn:

- [ ] Working Tree sauber ist.
- [ ] Tests grün sind.
- [ ] Build grün ist.
- [ ] DemoSeeder läuft.
- [ ] Desktop-Smoke-Test bestanden ist.
- [ ] Mobile-Smoke-Test bestanden ist.
- [ ] keine Blocker in Kernflows offen sind.
- [ ] bekannte MVP-Grenzen dokumentiert sind.
